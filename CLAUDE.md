# Godilla — правила проекта

## Текст в игре — только через HTML, не на холсте

**Правило.** Любой читаемый игровой текст (подсказки, всплывашки урона/очков, подписи
кнопок, цифры перезарядки, надписи в HUD и оверлеях) рисуем как **HTML-элемент**, а
**не** через `cx.fillText` на холсте.

**Почему.** Холст рисуется в «дизайнерском» разрешении 960×540 и растягивается на окно
дробным множителем (`Math.min(innerWidth/960, innerHeight/540)`, см.
[game/src/screen.js](game/src/screen.js)). Холст — растровая картинка, поэтому любой
текст, нарисованный на нём, при дробном увеличении мылится: жёсткие края глифов попадают
на дробные экранные пиксели и двоятся/размазываются («шакалится»). HTML же — векторный
шрифт, браузер масштабирует его резко при любом множителе (поэтому HUD всегда чёткий).

**Как.** Используем готовый слой [game/src/uitext.js](game/src/uitext.js):

- `floatText(x, y, txt, col, life)` — всплывающий текст (урон, очки): поднимается и
  гаснет, удаляется сам.
- `label(id, x, y, txt, col, size)` — постоянная подпись по ключу (создаётся один раз,
  дальше обновляется), `hideLabel(id)` — спрятать.

Координаты — те же «дизайнерские» (0..960 / 0..540), что и в игровой логике: слой лежит
внутри `#wrap` и масштабируется вместе с холстом.

Текст подсказок обучения — тем же принципом, отдельным HTML-элементом
(см. [game/src/tutorial.js](game/src/tutorial.js), `buildTextEl`).

**Тень текста** — только CSS `text-shadow` со сдвигом на целый «пиксель» шрифта и
**без полупрозрачностей** (вектор не двоится при масштабе).

**Исключение.** Чисто декоративные значки-глифы, не несущие читаемого смысла (например
`*` над оглушённым мобом), можно оставлять на холсте — это не «текст».

## Пауза всегда гасит тряску камеры

**Правило.** Когда мир ставится на паузу (обучение, диалог, выбор скилла, любое
модальное окно) — тряску камеры (`shake`) **обязательно обнуляем**.

**Почему.** `shake` затухает только внутри `update(dt)`. Если мир на паузе, `update`
не вызывается, и ненулевая тряска «застывает»: экран продолжает дёргаться под
неподвижной паузой. Это баг.

**Как.** В игровом цикле тряска гасится всегда, когда кадр не обновляется:

```js
if(running && !choosing && !tutorialFrozen() && !dialogueActive()) update(dt);
else shake = 0; // мир на паузе/стопе — тряски нет
```

То есть не нужно помнить про `shake = 0` в каждой точке паузы по отдельности — единый
`else` в цикле закрывает все случаи (и текущие, и будущие). Добавляешь новый вид паузы —
просто включаешь его в условие заморозки, и тряска погаснет сама.

## Сборка на itch.io — паковать ZIP только с прямыми слешами

**Правило.** Архив для загрузки на itch.io собираем **только** скриптом
[game/make-zip.ps1](game/make-zip.ps1) (после `npm run build`), а **не** через
PowerShell `Compress-Archive`.

**Почему.** `Compress-Archive` записывает пути внутри ZIP с обратными слешами
(`assets\index-….js`), а стандарт ZIP требует прямые (`assets/index-….js`). itch.io
такой архив распаковывает неправильно: файлы из `assets/` не попадают в подпапку, и
браузер получает на них **403 Forbidden**. Внешне это выглядит так: `index.html` (он в
корне) грузится, а игра — серый экран без логики (нет JS) и без оформления (нет CSS;
правило `.hidden` не действует → разом видны стартовый экран, настройки и пауза). Код
игры при этом исправен — диагноз был именно в упаковке (подтверждается в консоли
браузера: `GET …/assets/index-….js net::ERR_ABORTED 403`).

**Как.** `make-zip.ps1` кладёт каждый файл из `dist/` в архив с именем через прямой слеш
(`.Replace('\','/')`), `index.html` оказывается в корне. Проверка перед заливкой: пути в
архиве должны быть вида `assets/…`, а не `assets\…`.

**Дополнительно по странице игры на itch:** в Embed options ставим размер **960×540**
(родное разрешение холста), включаем **Fullscreen button** и **Automatically start on
page load** (иначе до клика видна серая заглушка). После перезаливки архива обновлять
страницу через **Ctrl+F5** — иначе браузер берёт старый кэш.

**Про `crossorigin`:** в [game/vite.config.js](game/vite.config.js) стоит плагин
`strip-crossorigin`, убирающий атрибут `crossorigin` из собранных тегов скрипта и стилей.
На текущую проблему он не влиял (она была в упаковке), но безвреден и убирает потенциальный
CORS-риск во вложенном окне itch — оставляем.
