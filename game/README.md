# Godilla — «Штурм горы»

Веб-игра (HTML5 Canvas, чистый JavaScript) под itch.io. Собирается через Vite.

## Запуск для разработки

```bash
cd Godilla/game
npm install      # один раз — поставить Vite
npm run dev      # запустит локальный сервер с горячей перезагрузкой
```

Открой адрес, который покажет Vite (обычно http://localhost:5173). Любая правка
в `src/` сразу применяется в браузере.

## Сборка для itch.io

```bash
npm run build    # соберёт готовую игру в папку dist/
```

Дальше на [itch.io](https://itch.io/game/new):
1. Заархивируй **содержимое** папки `dist/` в zip (внутри zip должен лежать
   `index.html` в корне, а не папка `dist`).
2. На странице игры: **Kind of project → HTML**, загрузи zip,
   поставь галку **«This file will be played in the browser»**.
3. Размер окна — 960×560 (как у канваса) или больше с запасом.

`npm run preview` — посмотреть собранную версию локально перед заливкой.

## Структура

```
src/
  config.js   — весь баланс игры (монстры, волны, прокачка, пороги урона). Крути тут.
  sprites.js  — спрайты. Сейчас рисуются из пиксельных карт; сюда придут PNG.
  audio.js    — звук. Сейчас синтезированные бипы; сюда придут настоящие звуки.
  game.js     — ядро: состояние, физика, цикл обновления и отрисовки, ввод, HUD.
  main.js     — точка входа.
assets/
  sprites/    — будущие картинки (PNG)
  sounds/     — будущие звуки
```

## Как добавлять контент

- **Поменять баланс** (ХП, скорости, волны, скиллы) → только `src/config.js`.
- **Заменить спрайт на картинку** → положи PNG в `assets/sprites/`, см. комментарий
  и заготовку `loadImage()` в `src/sprites.js`. Остальной код менять не нужно —
  он уже рисует объекты-картинки через `drawImage`.
- **Заменить звук** → положи файл в `assets/sounds/`, поправь тело нужной функции
  в объекте `sfx` в `src/audio.js`. Вызовы `sfx.grab()` и т.д. остаются как есть.

Исходный прототип сохранён в `../Prototype/demon-toss-2.html` как референс.
